# Story 1.4: 项目统一命名规范配置

**Epic:** Epic 1 - 项目脚手架与基础设施搭建
**Story ID:** 1.4
**Status:** in-progress

## User Story

As a developer,
I want consistent naming conventions across the project,
So that the codebase is maintainable and follows team standards.

## Acceptance Criteria

**Given** 项目代码已存在
**When** 应用统一命名规范
**Then** 包名统一为 `com.wherEver`
**And** Flutter模块前缀为 `wher_ever_`
**And** Java模块前缀为 `wher-ever-`
**And** 数据库表名前缀为 `we_`
**And** API路径前缀为 `/api/v1/`
**And** 配置文件已更新

## Technical Notes

### Naming Conventions

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | com.wherEver | com.wherEver.user |
| Flutter模块 | wher_ever_ | wher_ever_user |
| Java模块 | wher-ever- | wher-ever-user |
| 数据库表 | we_ | we_user |
| API路径 | /api/v1/ | /api/v1/user |
| 常量 | SCREAMING_SNAKE | MAX_RETRY_COUNT |
| 枚举值 | SCREAMING_SNAKE | SUBSCRIPTION_LEVEL_FREE |

### Quality Rules

- Checkstyle/ESLint 配置
- PMD/Flake8 配置
- Commit message 规范 (Conventional Commits)
