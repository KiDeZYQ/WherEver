# Story 2.5: 用户权限限制实施

**Epic:** Epic 2 - 用户账户与订阅系统
**Story ID:** 2.5
**Status:** in-progress

## User Story

As a developer,
I want to enforce permission checks throughout the app,
So that users can only access features allowed by their subscription level.

## Acceptance Criteria

**Given** Free用户尝试创建第4个提醒
**When** 点击创建按钮
**Then** 显示升级提示对话框
**And** 阻止创建行为

**Given** Premium用户
**When** 使用任何功能
**Then** 不受限制

**Given** 未登录用户
**When** 尝试访问受保护功能
**Then** 跳转到登录页

## Technical Notes

- Permission checks should be centralized in SubscriptionService
- UI should show disabled state for restricted features
- All feature creation should go through permission checks

## Dependencies

## Dev Agent Record

### Implementation Notes

**核心组件:**
- `PermissionGuard` - 权限守卫，整合AuthController和SubscriptionService
  - `check()` - 检查权限
  - `canCreateReminder()` - 提醒创建检查
  - `canCreateGeofence()` - 围栏创建检查
  - `canExportData()` - 数据导出检查
  - `showPermissionDialog()` - 显示权限对话框
- `PermissionResult` - 权限检查结果

**权限检查流程:**
1. Guest用户 -> 需要登录
2. Free用户 -> 检查limit，可能需要升级
3. Premium用户 -> 全部允许
