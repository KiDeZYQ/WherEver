# WherEver — 项目简介

> **版本：** v2（基于头脑风暴更新）
> **更新日期：** 2026-04-15

---

## 1. Executive Summary

WherEver 是一款基于**位置智能**和**时间上下文**的下一代提醒 App。与传统闹钟和待办工具不同，WherEver 的核心创新在于：提醒不再依赖用户主动触发，而是**跟随用户的物理位置和行为轨迹自动弹出**——"我到家时提醒我遛狗"，"我到达办公室时提醒我打卡"，"每天早上 8 点我还没出门就提醒我开会"。

现有的提醒工具（Google Reminders、Todoist、Any.do）有一个共同缺陷：**被动且死板**。它们要求用户在固定时间打开 App、手动查看清单、逐项处理。WherEver 的目标是让**提醒主动找到用户**，在最需要的时刻、最恰当的地点弹出，从根本上改变人获取任务的方式。

WherEver 定位为 Android 平台**最智能的情境感知提醒工具**，采用免费增值（Freemium）商业模式。

---

## 2. The Problem

### 2.1 现有方案的三大局限

**死板一：时间触发无法适应变化的生活**
- 闹钟只能在预设时间响起，但人的生活节奏是动态的
- "每天 8 点开会"——但如果今天休息呢？如果今天在外地呢？固定时间提醒让人被时间绑架

**死板二：地理位置提醒几乎是空白或不可靠**
- Google Assistant 支持"到家时提醒我..."，但能力极其基础
- Any.do 的位置提醒被大量用户反馈定位不准、通知延迟
- Todoist 根本不提供位置触发功能
- Android 后台进程被系统杀死后，位置触发完全失效——这是 Android 平台的系统性挑战

**死板三：免费版把核心价值锁在付费墙后**
- 位置触发通常是 Premium 专属功能，但付费后体验仍不稳定
- 用户为"可靠性"付费，但可靠性不达标

### 2.2 真实用户场景的痛苦

> "我在超市门口，但我完全忘了要买什么——因为我的购物清单在手机里，我得掏出来、解锁、打开 App 才能看到。如果在我接近超市时提醒自动弹出来，就不会忘。"

> "每天设了 10 个闹钟，但真正需要提醒的事情根本不是时间点——是我到某个地方时该做什么。"

> "最烦的是，有些提醒设置了但系统根本不触发，久了就不信任这些工具了。"

### 2.3 根因洞察（Five Whys 探索结论）

> **用户需要的不只是提醒功能，是一个"永远不会让我漏掉、永远不会让我尴尬"的信任系统。**

这不是一个功能需求，而是**心理需求**——用户不信任自己的记忆，所以需要外部系统来确保安全。WherEver 的所有设计都必须围绕"信任感"这个核心情绪。

## 3. The Solution

### 3.1 核心能力矩阵

| 能力 | 描述 |
|------|------|
| **对话式提醒创建** | 用户通过自然语言描述提醒需求，AI 解析后展示结构化确认卡片，经用户确认后添加。支持追问补全和多轮修正 |
| **三重触发引擎** | 支持三种独立的触发模式：① 进入触发（arrive）② 离开触发（leave）③ **出发前触发（departure）**——在用户还没到达目的地时提前预警，结合日历/天气等情境信息预防问题 |
| **位置触发引擎** | 基于地理围栏（Geofencing），支持进入/离开双模式，兼容后台持久运行 |
| **时间+位置组合触发** | 可以设置"工作日早上 8 点，我还没到公司就提醒开会"——条件叠加逻辑 |
| **智能重复** | 支持"每 X 天/周在特定位置触发一次"、"直到确认才停止重复"等高级规则 |
| **情境感知** | 结合日历、天气、交通状况调整提醒时机（Premium） |
| **主动推送** | 利用 Android 前台服务和 AlarmManager 保障后台存活，解决被系统杀死的痛点 |
| **地图直连导航** | 提醒卡片上显示地点，点击直接唤起高德/百度地图导航，无需复制粘贴地址 |
| **轻分享** | 约饭/会议等场景可分享给赴约对象，双方均收到同一提醒并可一键导航 |

### 3.2 触发模式详解：Departure（出发前触发）

三种触发模式的核心区别：

| 触发模式 | 何时触发 | 典型场景 |
|---------|---------|---------|
| **Arrive** | 用户进入围栏区域 | "我到超市了，提醒我买牛奶" |
| **Leave** | 用户离开围栏区域 | "我离开家了，提醒我锁门" |
| **Departure** | 用户尚未出发，目标时间临近 | "今天有雨，出发前提醒我带伞" |

**Departure 是 WherEver 的独特创新**——它不是在问题发生时才提醒，而是在问题发生**之前**就预警。典型应用：
- 天气预警：日历有户外活动 + 天气预报有雨 → 出发前提醒带伞
- 交通预警：会议临近但用户还在路上 → 出发前提醒迟到风险
- 物品提醒：日历有出差日程 → 出发前提醒检查行李

### 3.3 订阅分层设计

#### Free（免费版）
- 最多 **5 个活动提醒**
- 仅支持**纯时间触发**（标准闹钟/定时提醒）
- 1 个位置围栏（最多 100m 半径）
- 最近 7 天历史记录
- **对话式创建（基础）**：自然语言输入 → 结构化确认卡片，限单轮确认
- 有广告

#### Premium（¥12/月 或 ¥96/年）
- **无限提醒和位置围栏**
- 完整的**三重触发引擎**（进入/离开/出发前，自定义半径）
- **时间+位置组合条件**（条件叠加）
- **出发前情境感知**（日历+天气+交通）
- 高级重复规则（间隔重复、位置触发去重）
- 最近 **30 天历史记录 + 数据导出（CSV/JSON）**
- **对话式创建（完整）**：支持多轮追问、条件追加、异常修正
- **无广告**
- 优先客户支持

#### Pro（¥25/月 或 ¥200/年）— AI 层
- Premium 全部功能
- **AI 智能调度**：基于用户历史确认行为，自动调整提醒时间
- **预测性提醒**：结合日历和位置历史，提前预判用户可能忘做的事
- **高级数据分析**：热力图（常去地点）、提醒响应率、遗忘模式分析
- **自定义围栏库**：保存常用地点为模板

### 3.4 交互设计：动效语言系统

三种状态的动效语言，让用户无需阅读文字就知道发生了什么：

| 状态 | 动效 | 情绪传递 |
|------|------|---------|
| **触发时** | 卡片从底部滑入 + 轻微放大（pop）+ 一次短震动 | 紧迫感、存在感——"这事来了，你现在处理" |
| **确认后** | 卡片向上滑出 + 渐隐 + 轻微 success 震动 | 平静、完成感——"好了，这件事结束了" |
| **推迟后** | 卡片轻推一下（shake）然后安静地缩回原位，无震动 | 淡定——"知道了，等会儿再来找你" |

### 3.5 界面设计语言

**视觉方向：金融 App 卡片式布局**

借鉴高端金融 App 的设计语言——深色/微妙渐变底色 + 圆角卡片 + 精致动效，让每条提醒像一张有分量的"任务卡"。

- 差异化于现有效率工具（Todoist 列表式）偏功能性界面
- 传递"信任感"和"品质感"，呼应用户的心理需求
- 卡片即承载地点信息、可交互地图、导航入口

### 3.6 为什么 WherEver 能解决"死板"问题

传统工具是**被动查询**：用户主动想起来 → 打开 App → 看到清单 → 处理事务。

WherEver 是**主动推送**：系统在正确的时刻 → 正确的地点 → 弹出正确的提醒 → 用户确认或推迟。

核心逻辑改变带来三个根本差异：
1. **零认知负荷**：不需要想着"我待会儿要记得做什么"，围栏自动触发
2. **上下文精准**：在超市门口弹购物提醒，比在家想起更有效
3. **动态适应**：AI 层会学习用户行为，从"固定规则"进化到"个性化预测"

## 5. Who This Serves

### 5.1 主要用户画像

**都市通勤者（18-35 岁）**
- 特点：日常在多个地点之间切换（家→公司→健身房→超市）
- 痛点：不同地点有不同待办，但现有工具无法根据地点区分；怕漏掉重要事情被老婆/同事说
- 成功定义：不再漏掉任何"到某地该做的事"，建立对系统的完全信任

**家庭事务管理者（25-45 岁）**
- 特点：需要记住购物、接送孩子、吃药等多种提醒
- 痛点：事项太多，全部靠脑子记压力巨大
- 成功定义：信任系统，知道它会在正确时机提醒

### 5.2 次要用户

- 轻度 ADHD 用户（需要外部记忆辅助）
- 差旅频繁的商务人士（地点不固定但事务固定）

## 7. MVP Scope

### 7.1 MVP 功能定义

**MVP = 最小可行产品，验证核心价值假设**

**核心价值假设：** "用户愿意为可靠的位置触发提醒 + 地图导航体验付费"

**Must Have（第一版必须做）：**

| 功能 | 描述 | 优先级理由 |
|------|------|-----------|
| **对话式提醒创建** | 自然语言 → 结构化确认卡片 | 降低创建门槛，差异化体验 |
| **三重触发引擎** | Arrive / Leave / Departure 三种模式 | 核心差异化，Departure 独有 |
| **地图直连导航** | 卡片点击唤起高德/百度地图 | 从"告诉你去哪"到"带你去那" |
| **金融 App 卡片式 UI** | 质感设计 + 深色主题 | 信任感建立，差异化视觉 |
| **动效语言系统** | 触发（pop+震动）/确认（渐隐）/推迟（轻推） | 让用户无需看字就知道发生了什么 |

**Nice to Have（第二版）：**

| 功能 | 描述 | 移出原因 |
|------|------|---------|
| 日历+天气出发前提醒 | 需要天气 API + 日历打通，复杂度高 | MVP 聚焦位置触发，情境感知可后加 |
| 约饭/会议轻分享 | 需要账号体系+分享逻辑 | 先做透单用户场景 |

**明确排除：**
- AI 预测层（Pro 版本，第二阶段）
- 跨设备同步（第二阶段）
- Apple Watch / 可穿戴设备整合（第三阶段）
- 团队协作功能（无限期排除——个人用户工具定位）

### 7.2 三阶段路线图

| 阶段 | 内容 | 验证假设 |
|------|------|---------|
| **Phase 1（MVP）** | 三重触发引擎 + 对话创建 + 卡片 UI + 动效 + 地图导航 | 位置触发提醒是否有人用？付费意愿？ |
| **Phase 2（Growth）** | 出发前情境感知（天气/交通）+ 约饭轻分享 + 自定义围栏库 | 用户是否愿意为"预防性提醒"付费？ |
| **Phase 3（Expansion）** | AI 智能调度 + Pro 数据分析 + 跨设备同步 + Widget | WherEver 能否成为真正的"第二大脑"？ |

## 8. Vision

WherEver 的愿景是成为每台 Android 手机上的**第二大脑**——一个无处不在、从不遗忘的数字记忆伴侣。

**未来 2-3 年目标状态：**

- WherEver 理解你的生活轨迹，知道你每天去哪里、待多久、需要做什么
- AI 不仅能提醒，还能主动预测："你明天要去北京出差，要不要提前提醒你整理行李？"
- 成为 Android 平台情境感知提醒的代名词，像"Google"在搜索领域的地位一样
- 从个人工具延伸到轻度家庭场景（如家庭位置共享提醒）

**长期命题**：如果智能手机是人的外部大脑，WherEver 正在成为这个大脑的**主动记忆系统**——不是等你想起来，而是帮你记住，在最需要的时刻。

## 附录：待确认事项（用于后续 PRD 阶段）

以下问题需要在 PRD 制定时与利益相关者明确：

1. **目标发布时间**：MVP 计划什么时候上线？
2. **技术选型**：React Native / Flutter / Native Android？是否有现有团队？

7. **对话式创建技术方案**：本地 LLM（TensorFlow Lite）还是云端 API？影响隐私策略和离线体验
